# 安卓电视高清影音APP - 产品需求说明

## 1. 项目概述与核心目标

开发一款专适配安卓智能电视、电视盒子（TV Box）的综合性影音聚合软件。提供流畅的高清电视直播（Live TV）和丰富的点播（VOD）资源（涵盖电影、电视剧、动漫、海外剧、地方戏曲）。要求操作极简、焦点清晰，完美适配遥控器操作。

## 2. 核心交互与 UI 规范 (TV 端专属重点)

### 遥控器适配 (D-pad Navigation)
- 所有界面必须支持遥控器"上下左右"切换焦点，"确认"键进入，"返回"键退出
- 绝对不能出现需要鼠标指针才能点击的死角

### 焦点高亮 (Focus State)
- 当前选中的按钮、海报必须有明显的放大效果、发光边框或高亮底色
- 确保用户坐在沙发上能清楚知道当前选中的位置

### 横屏大屏 UI
- 采用标准的 Android TV Leanback 风格或瀑布流布局
- 字体要大（至少 18sp 以上），对比度要高
- 背景以深色模式（Dark Mode）为主，减少大屏刺眼感

## 3. 核心功能模块划分

### 模块一：高清电视直播 (Live TV)

#### 频道分类
- 左侧呼出分类菜单（如：央视、卫视、地方台、少儿频道、体育频道）

#### 无缝换台
- 支持遥控器"上下键"快速切换频道
- 支持数字键直接输入频道号换台

#### EPG 节目单
- 播放时按下遥控器"右键"或"菜单键"，可显示当前频道的节目预告时间表

#### 解码与容错
- 支持软硬解切换
- 如果当前高清源（m3u8/flv）卡顿或失效，系统需支持自动切换到备用直播源

### 模块二：影视点播大厅 (VOD)

#### 精细化分类频道
- 电影 / 电视剧 / 动漫
- 海外专区：单独设立"美剧"、"英剧"板块
- 戏曲专区：单独设立"豫剧"、"京剧"等板块，界面设计可适当加大封面和字体

#### 瀑布流展示
- 首页按分类提供瀑布流推荐（最新上线、热门推荐、高分经典）

#### 详情页展示
- 选中某部影片后，展示大图海报、豆瓣评分、年份、主演、剧情简介
- 展示"选集"列表

### 模块三：全局搜索功能 (Global Search)

#### T9 键盘与全拼/首字母
- 电视端输入极不方便，必须提供 TV 专用的软键盘（T9 或全键盘）
- 核心支持首字母搜索（例如搜"狂飙"，输入"KB"即可）

#### 跨库搜索
- 一次搜索，同时返回电影、电视剧、戏曲等所有相关匹配结果

#### 语音搜索 (进阶)
- 如果设备支持，接入 Android TV 的原生语音输入
- 按住遥控器语音键说出"想看豫剧朝阳沟"即可直达

### 模块四：播放器体验 (Player Experience)

#### 画质选择
- 菜单内支持切换分辨率（720P, 1080P, 4K）

#### 进度控制
- 遥控器"左右键"控制快进/快退
- 显示带时间轴的进度条缩略图

#### 字幕与音轨
- 针对美剧/英剧，支持内嵌字幕和外挂字幕切换
- 支持中英双语字幕显示
- 支持多音轨切换

#### 记忆播放
- 自动记录上一次观看的集数和时间点
- 下次打开直接续播

### 模块五：离线缓存与下载管理 (Offline Downloading & Storage)

#### 存储路径与外接设备管理
- 多存储介质识别：系统需自动检测电视的内部存储以及插入的外部存储设备（U盘、移动硬盘、SD卡）
- 首选下载路径设置：在软件的"设置"界面中，提供"下载路径选择"功能。如果检测到外接 U 盘，强烈建议引导用户将默认下载路径设为外接设备
- 拔插保护：当正在下载或播放缓存视频时，如果外接 U 盘被拔出，系统需弹出明显的 Toast 提示，并自动暂停所有相关下载任务，防止程序崩溃

#### 电视端专属的缓存交互设计
- 一键缓存/选集缓存：在影视详情页提供"下载/缓存"按钮。点击后弹出的选集列表必须完美适配遥控器焦点
- 支持"单集选中下载"
- 支持"一键缓存全集"（针对集数较少的剧集）
- 画质选择选项：在开始下载前，提供清晰度选择（标清 480P、高清 720P、超清 1080P），并在选项旁预估文件大小

#### 下载管理中心 (Download Center UI)
- 独立入口：在首页侧边栏或主导航栏提供显眼的"我的下载"或"本地缓存"入口
- 存储空间进度条：在下载管理页面显示存储空间进度条（如：已用 14GB / 总共 16GB），并在空间不足（如低于 10%）时标红警告
- 任务管理列表：显示正在下载的任务列表（包含网速展示、进度条、百分比）
- 支持遥控器选中某个任务后，按"确认键"弹出操作菜单：【暂停】、【继续】、【删除】
- 支持"全部暂停"和"全部开始"快捷操作

#### 智能清理与空间管理
- 阅后即焚（可选开关）：在设置中提供"看完自动删除已缓存剧集"的开关
- 空间不足预警：当剩余空间不足以完成当前下载任务时，暂停下载并弹出全屏对话框提醒用户

## 4. 技术栈与架构约束

### 开发语言
- Kotlin / Java

### UI 框架
- 推荐使用 Android 原生的 Leanback 库
- 或基于 RecyclerView 深度定制的 TV 瀑布流组件
- 跨平台方案：Flutter for TV（需注意遥控器焦点调优）

### 播放器内核
- ExoPlayer（谷歌官方，对流媒体支持极好）
- 或 IjkPlayer（B站开源，解码能力强）
- 不要使用系统默认的 MediaPlayer

### 数据接口
- 预留好 RESTful API 接口或解析规则（JSON/XML）
- 方便后期接入影视CMS后台（如苹果CMS、海洋CMS等）或 IPTV 播放源

### 下载引擎
- 强烈建议使用 ExoPlayer 自带的 DownloadManager
- 支持分片下载、断点续传
- 对 HLS (m3u8) 和 DASH 流媒体格式的缓存支持极其优秀

### 后台机制
- 使用 Android 的 WorkManager 或 Foreground Service
- 保证下载任务在用户切换到其他 App 时依然能稳定运行，不会被系统轻易杀掉

### U盘权限 (SAF)
- 使用 Storage Access Framework (SAF) 标准 API 来申请U盘的读写权限
